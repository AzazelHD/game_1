New this session — SettingsRowWindow (unification, in progress)

New reusable UIWindow subclass: ui/windows/SettingsRowWindow.h/.cpp. Supports three row types — Button, Setting (label + left/right-adjustable value, e.g. resolution/window mode), Slider (label + embedded Slider* + live left/right step). Up/Down navigates focus internally; Left/Right adjusts the focused row's value; Accept activates a Button row. Emits real UIEvents (ActionSelected/ActionCanceled) via the standard emit() pattern — no direct callbacks into the owning Scene, consistent with the established ActionId boundary rule (independent UIWindow ↔ owning class communication only via events; per-row local logic like onAdjust/getDisplayValue stays as lambdas since that's not crossing the boundary).
New ActionId::AdjustLeft/AdjustRight added — confirmed correct usage per the established rule (emitted by an independent window, not a BattleMenuItem-style internal construct).
New WindowId::SettingsGraphics, SettingsAudio, SettingsExitConfirm added (SettingsMenu already existed, reused for the hub page).
AudioSettings fully converted to push a SettingsRowWindow via its own UIManager member, react to events in processUIEvents(), and reuse ConfirmWindow for the unsaved-changes exit popup (replacing the old hand-drawn popup). Confirmed and fixed a real bug found during conversion: AudioSettings::handleInput() had a guard (if (m_showExitConfirm) return;) that would have silently blocked all input to the pushed ConfirmWindow, soft-locking the exit dialog — removed; UIManager already correctly routes input only to the topmost visible window on its own.
GraphicsSettings not yet converted — still on the old hand-rolled row rendering. Next concrete step for this thread.
Layout math ported and fixed, not simplified: SettingsRowWindow's row layout (label box + controls box, each self-sized via HorizontalLayout::computeBounds/layoutRow from real measureText() widths, not fixed percentages) is a faithful port of the old monolithic SettingsState::computeAudioRowLayout(), generalized to also support non-slider Setting rows. Three real bugs found and fixed against a first draft that used fixed 40/60% splits:

Label/slider overlap and "%" text landing outside the row box — root cause was abandoning measured-text sizing for hardcoded percentages; fixed by porting the original HorizontalLayout::Container-based approach.
Master Volume and Music Volume rows having mismatched label-column widths — root cause was measuring each row's label individually instead of once across all rows and sharing the max width (which the original code did on purpose, for column alignment). Fixed via a new measureMaxLabelWidth() helper, computed once per render() call and passed into computeRowLayout() for every row.
Slider vertically off-center in its row — root cause was a leftover fixed kSliderMarginTop (16px) copied from the old code without re-deriving it; replaced with a computed centering offset (rowH - kTrackH * ui) * 0.5f. kSliderMarginTop constant now unused, safe to delete.


Row outer padding done properly, not hand-hacked: user explicitly rejected a first-pass manual x - gap/width + gap*2 fix for row-edge spacing in favor of using the existing Insets/HorizontalLayout::computeBounds system consistently — final version wraps the row's content in a HorizontalLayout::Container with Insets{0, kRowSideGap*ui, 0, kRowSideGap*ui} padding, matching how every other box in the file is already computed, rather than a one-off arithmetic adjustment.
